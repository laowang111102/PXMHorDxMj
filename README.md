# 车牌识别系统介绍 python1112

## 项目概述

车牌识别系统是一个基于Python和Django框架开发的Web应用程序，采用前后端分离的架构设计。该系统主要提供车牌图片的上传识别功能，并对识别后的车牌信息进行管理。

## 技术栈

- **架构**: B/S、MVC
- **系统环境**: Windows
- **开发环境**: PyCharm、Python 3.x、MySQL 5.x
- **技术栈**: Python、Django、MySQL

## 功能模块

### 系统角色

系统设定了两个主要角色：用户和管理员。

#### 前台系统

- **首页**: 展示系统的基本信息和导航。
- **登录**: 用户和管理员登录系统。

#### 后台管理系统

##### 管理员端

- **车牌识别**: 管理员可以上传车牌图片进行识别，系统返回车牌号及颜色。
- **车牌管理**: 管理识别后的车牌信息，包括查看、修改和删除。
- **用户管理**: 管理用户信息，包括添加、修改和删除用户。

##### 用户端

- **车牌识别**: 用户上传车牌图片进行识别，系统返回车牌号及颜色。
- **车牌管理**: 用户查看和管理自己的车牌识别记录。

## 运行环境

- 操作系统：Windows
- Python版本：Python 3.x
- 数据库：MySQL 5.x

## 目录结构

```
项目根目录/
│
├── apps/               # 应用代码目录
│   ├── admin/          # 管理员端应用
│   └── user/           # 用户端应用
│
├── db/                # 数据库相关文件
│
├── media/             # 存储上传的媒体文件
│
├── static/            # 静态文件目录
│
└── templates/         # 模板文件目录
```

## 部署步骤

1. 准备Python 3.x和MySQL 5.x运行环境。
2. 克隆或下载项目代码至服务器。
3. 安装项目依赖，使用`pip install -r requirements.txt`。
4. 配置数据库，导入数据库结构和初始数据。
5. 收集静态文件，使用`python manage.py collectstatic`。
6. 运行Django应用，使用`python manage.py runserver`。
